# つべナビ デザイン調整ガイドライン 🛠️

このガイドラインは、QAレビューとユーザーフィードバックに基づくデザイン調整を行う際の指針です。デザインの一貫性を保ちながら、効果的に改善を進めるための参考にしてください。

## 📊 調整プロセス

### 1. フィードバック分析
- QAチェックリストの結果とユーザーフィードバックを集計・分析
- 主要な問題点と改善要望をリスト化
- 優先度の決定（重要度と影響範囲による）

### 2. 改善計画の策定
- 問題ごとに対応策を検討
- 変更によるブランドイメージへの影響を評価
- 実装の難易度と工数を見積もり

### 3. 調整の実施
- 変更を段階的に実施（一度に全体を変更しない）
- 各変更後のテストと検証
- 開発チームとデザインチームの連携確保

### 4. 検証と評価
- 調整後の再テスト
- ステークホルダーによるレビュー
- 必要に応じてさらなる調整

## 🎯 調整の優先順位

1. **機能性に影響する問題**
   - ユーザビリティを阻害する問題
   - 操作性に直接影響する要素

2. **ブランド一貫性の問題**
   - ロゴやカラーの使用が一貫していない箇所
   - デザイン言語の統一性が欠けている部分

3. **視覚的な調整**
   - レイアウトの微調整
   - 余白やアライメントの最適化

4. **エンハンスメント（機能追加）**
   - 新たなデザイン要素の追加
   - 視覚効果の強化

## 🎨 デザイン変更のガイドライン

### ブランドカラー
- **基本ルール**: プライマリーカラー (#FF3366)、セカンダリーカラー (#3A86FF)、アクセントカラー (#FFD60A) の比率は7:2:1を目安に
- **許容範囲**: 各カラーの明度・彩度の調整は±10%まで
- **避けるべき変更**: 全く新しいカラーの追加、基本カラーの完全な置き換え

### タイポグラフィ
- **フォントファミリー**: Montserrat、Inter、Poppinsの3種類は維持
- **サイズ調整**: 既存の階層を維持しながら、必要に応じて各サイズを±2px以内で調整可能
- **行間・文字間**: 視認性向上のために微調整可（行高は1.2〜1.6の範囲で）

### レイアウト
- **グリッドシステム**: 基本の12列グリッドを維持
- **余白調整**: コンポーネント内の余白は8の倍数（8px, 16px, 24px, 32px...）で調整
- **アライメント**: テキストと要素のアライメントは一貫性を保つ

### コンポーネント
- **ボタン**: 基本形状とサイズ比は維持、細部のみ調整可能
- **カード**: 角丸、影、間隔のスタイルは一貫して維持
- **フォーム要素**: 入力フィールドの高さと内部余白は統一

## 🛠️ 技術的実装のガイドライン

### CSS変数
```css
/* 変更可能な範囲の例 */
--card-border-radius: 8px; /* 6px〜10pxの範囲で調整可 */
--button-padding: 0.5rem 1rem; /* 比率を維持しながら調整可 */
--transition-speed: 0.2s; /* 0.15s〜0.3sの範囲で調整可 */
```

### レスポンシブブレークポイント
- 基本ブレークポイントの値は変更しない
  - sm: 640px
  - md: 768px
  - lg: 1024px
  - xl: 1280px
  - 2xl: 1536px

### アニメーション
- トランジション: ease-in-outの基本イージングを維持
- アニメーション時間: 0.2s〜0.5sの範囲に収める
- 過度なアニメーションの追加は避ける

## 📝 調整記録の管理

すべての調整は以下の形式で記録してください：

```
## [日付] 調整内容

### 変更点
- 変更した要素とその詳細

### 理由
- 変更を行った理由またはフィードバック元

### 影響範囲
- 変更による影響を受ける部分

### スクリーンショット
- 調整前後の比較画像
```

## 👥 調整の承認フロー

1. デザイナーによる調整案の作成
2. 開発チームとの技術的実現性の確認
3. プロジェクトマネージャーによる承認
4. 実装とレビュー
5. 本番環境への反映
