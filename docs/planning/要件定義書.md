# つべナビ 要件定義書 📝

## 1. プロジェクト概要 🌟

### 1.1 背景と目的
つべナビは、YouTubeコンテンツクリエイターや運営者向けに、チャンネルの基本情報や動画リストを取得し、データの整理や基本的な分析をサポートするWebアプリケーションです。チャンネルURLを入力するだけで、簡単にデータ収集ができ、効率的なチャンネル管理を実現します。

### 1.2 ターゲットユーザー
- YouTubeコンテンツクリエイター
- チャンネル運営マネージャー
- マーケティング担当者
- コンテンツ制作チーム

### 1.3 ビジネスモデル
- **無料ツール**: 現段階では基本機能を無料で提供
- **将来計画**: 追加機能や高度な分析ツールの開発を検討中

## 2. 機能要件 ⚙️

### 2.1 YouTube API連携
- YouTube API Keyの設定と管理
  - クライアントサイドでのAPI Key保存
  - API Key検証機能
- チャンネルURL/IDからのデータ取得

### 2.2 ダッシュボード機能
- チャンネル検索フォーム
  - URL入力とバリデーション
  - エラーハンドリング
- アプリ説明と使い方ガイド
- API設定リマインダー

### 2.3 チャンネル基本情報表示
- チャンネル統計データの表示
  - チャンネル名、登録者数、総再生回数
  - チャンネルアイコン表示
  - チャンネル説明文表示
- チャンネルタグ表示
- チャンネルアイコン保存機能

### 2.4 動画リスト機能
- 動画一覧表示
  - タイトル、公開日、再生数、高評価数、コメント数
  - サムネイル表示
  - 動画リンク
- ソート機能（再生数、公開日など）
- 期間別フィルタリング
  - 全期間/1週間/1ヶ月/3ヶ月/半年/1年

### 2.5 エクスポート機能
- Excel形式でのデータエクスポート
  - チャンネル情報シート
  - 動画データシート
  - カスタマイズオプション
- サムネイル一括ダウンロード
  - ZIP形式でのダウンロード
  - 進捗表示

### 2.6 表示設定
- ダークモード切替
- レスポンシブ表示対応
- エクスポート設定カスタマイズ

### 2.7 チャンネル分析機能 【未実装 - 将来の開発予定】
- チャンネル統計データの表示
  - 登録者推移
  - 総再生時間
  - 平均視聴維持率
- 視聴者層分析
  - 地域分布
  - 年齢・性別分布
  - アクセスデバイス分析
- 成長トレンドの可視化
- エンゲージメント率の分析

### 2.8 動画分析機能 【未実装 - 将来の開発予定】
- 動画一覧表示（ソート・フィルター機能）
- 個別動画詳細分析
  - 再生数推移
  - 視聴者維持率グラフ
  - エンゲージメント指標（高評価、コメント、共有）
  - トラフィックソース分析
- 動画間パフォーマンス比較
- タグ・キーワード分析
- 最適な公開時間分析

### 2.9 レポート生成機能 【未実装 - 将来の開発予定】
- レポートテンプレート選択
- カスタムレポート設定
- 日付範囲選択
- データ可視化オプション
- PDF形式でのエクスポート

### 2.10 設定管理 【未実装 - 将来の開発予定】
- アカウント設定
- YouTube API連携設定（現在はローカル保存のみ）
- 通知設定

- プラン管理・支払い設定

### 2.11 管理者機能 【未実装 - 将来の開発予定】

- システム状態モニタリング

## 3. 非機能要件 🛠️

### 3.1 UI/UX要件
- レスポンシブデザイン（モバイル・タブレット・デスクトップ対応）
- モダンでクリーンなインターフェース
- ダークモード対応
- 直感的な操作性
- アクセシビリティへの配慮

### 3.2 パフォーマンス要件
- ページロード時間：2秒以内
- APIレスポンス時間：1秒以内
- 同時ユーザー接続：最低1000人
- 大量データ処理時のスムーズな動作

### 3.3 セキュリティ要件
- HTTPS通信
- JWTによる認証
- パスワードの安全な保存（ハッシュ化）
- API呼び出しの認証・認可
- XSS/CSRF対策
- レート制限の実装

### 3.4 可用性要件
- 稼働率99.9%以上
- 計画的なメンテナンス時間の設定
- 障害発生時の迅速な復旧体制

### 3.5 拡張性要件
- マイクロサービスアーキテクチャの採用
- スケーラブルなバックエンド設計
- 新機能追加のための柔軟な構造

## 7. 制約条件と前提 ⚠️

### 7.1 制約事項
- YouTube API の利用制限（クォータ）への対応
- ブラウザ互換性（IE11をサポート対象外とする）
- モバイルデバイスの画面サイズ制約

### 7.2 前提条件
- ユーザーがYouTubeチャンネルへの適切なアクセス権を持っていること
- インターネット接続が利用可能であること
- YouTube APIの仕様変更に対応すること

## 9. リスクと対策 🛡️

### 9.1 想定リスク
- YouTube API仕様変更によるシステム影響
- データ量増加によるパフォーマンス低下
- セキュリティ脆弱性の発生

### 9.2 対策
- APIラッパー層の実装によるAPI変更の影響局所化
- スケーラブルなデータ処理アーキテクチャの採用
- 定期的なセキュリティ監査と脆弱性テスト
