# 開発コンテキスト管理

このディレクトリは、プロジェクトの開発コンテキスト情報を管理するための場所です。

## ディレクトリ構造

```
context/
└── README.md                # このファイル（コンテキスト管理の説明）

docs/issues/                 # 実際のIssue管理ディレクトリ
├── #1250/                   # Issue番号ごとのディレクトリ
│   ├── 000_about.md         # Issueの概要
│   ├── 001_task.md          # タスクリスト
│   └── DevLogs/             # 実装ログディレクトリ
│       └── 250419.md        # 日付ごとの実装ログ
├── #1251/                   # Issue番号ごとのディレクトリ
│   ├── 000_about.md         # Issueの概要
│   ├── 001_task.md          # タスクリスト
│   └── DevLogs/             # 実装ログディレクトリ
│       └── 250419.md        # 日付ごとの実装ログ
└── template_devlog.md       # 実装ログのテンプレート
```

## 運用方法

### 新しいIssueの作業を開始する場合

1. `docs/issues/#XXXX` ディレクトリを作成（XXXXはIssue番号）
2. `000_about.md` にIssueの概要を記載
3. `001_task.md` にタスクリストを作成
4. `DevLogs` ディレクトリを作成

### 実装作業を記録する場合

1. `docs/issues/#XXXX/DevLogs` ディレクトリ内に `YYMMDD.md` ファイルを作成（YYMMDDは年月日）
2. `docs/issues/template_devlog.md` を参考に、以下の内容を記録：
   - ユーザーからの指示内容
   - 実装した内容の詳細
   - 編集したファイルとコード
   - 次に行うタスク

## 実装ログの書き方

実装ログには以下の内容を含めてください：

1. **タスク**: 取り組んでいるIssue番号とタイトル
2. **ユーザー指示**: ユーザーからの指示内容をそのまま記載
3. **実装内容**: 実際に行った作業の詳細（ファイル編集、コマンド実行など）
4. **次のタスク**: 次に予定されている作業
5. **メモ・備考**: 気づいた点や改善提案など（任意）

## 注意事項

- 実装ログは日々の作業履歴として残すため、実際に行った作業だけを記録してください
- コードスニペットを含める場合は、言語名を指定してシンタックスハイライトを有効にしてください
- 機密情報（APIキーなど）が含まれないよう注意してください
