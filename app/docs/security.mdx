# セキュリティ対策

つべナビでは、お客様のデータを保護するために以下のセキュリティ対策を実施しています。

## 基本的なセキュリティ対策

### 1. API Keyの安全な管理

つべナビでは、YouTube Data APIにアクセスするために必要最小限の権限を持つAPI Keyのみを使用しています。

- API Keyはブラウザのローカルストレージに保存され、サーバーには送信されません
- HTTPリファラー制限を推奨し、第三者によるAPI Keyの不正使用を防止
- API Keyに設定できる権限は読み取り専用のみで、チャンネルの変更や動画のアップロードなどの操作はできない設計

### 2. データ暗号化

- **通信暗号化**: すべての通信はTLS 1.3（SSL）によって暗号化されています
- **保存データの暗号化**: サーバー上のデータはAES-256による暗号化で保護されています
- **キー管理**: 暗号化キーはHSMで厳重に管理されています

### 3. アクセス制御

- ユーザー認証には多要素認証（MFA）オプションを提供
- セッション管理の厳格化（一定時間操作がない場合の自動ログアウト）
- IPアドレスベースのアクセス制限機能（チーム利用向け）

## 定期的なセキュリティ対策

### 1. セキュリティ監査

- 四半期ごとの内部セキュリティ監査の実施
- 年1回の外部専門機関によるペネトレーションテスト
- 脆弱性スキャンによる継続的なセキュリティチェック

### 2. ソフトウェアの更新

- セキュリティパッチの迅速な適用
- 依存ライブラリの定期的な脆弱性チェックと更新
- インフラストラクチャのセキュリティ強化

### 3. セキュリティトレーニング

- 開発チームへの定期的なセキュリティトレーニング
- インシデント対応訓練の実施
- セキュリティベストプラクティスの継続的な見直し

## インフラストラクチャのセキュリティ

### 1. クラウドセキュリティ

- クラウドプロバイダのセキュリティベストプラクティスの導入
- ネットワークセグメンテーションの実施
- ファイアウォールとWAF（Web Application Firewall）の導入

### 2. バックアップと災害復旧

- データの定期的なバックアップ
- 複数リージョンでの冗長化
- 災害復旧計画の定期的なテスト

### 3. モニタリングと検知

- 24時間365日のセキュリティモニタリング
- 異常検知システムの導入
- インシデント対応プロセスの確立

## コンプライアンス

- GDPR（EU一般データ保護規則）への準拠
- CCPA（カリフォルニア州消費者プライバシー法）への対応
- 定期的なプライバシー影響評価の実施
